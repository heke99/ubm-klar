# Municipal Onboarding

UBM Klar onboarding must guide each municipality from initial setup to production readiness.

## Stages

1. Organisation
   - municipality profile
   - committees
   - departments
   - units
   - system owner
   - DPO
   - information security officer
   - legal contact
   - UBM contact
   - finance contact

2. Deployment
   - choose Model B or Model C
   - configure test/stage/prod
   - configure domains
   - configure storage
   - configure backup and restore
   - configure SIEM
   - configure support access model

3. Authentication
   - Entra ID / OIDC / SAML
   - MFA verification
   - group mapping
   - role mapping
   - break-glass accounts

4. Source Systems
   - LSS system inventory
   - economic assistance system inventory
   - economy/payment system inventory
   - document system inventory
   - archive system inventory
   - import methods
   - data owners

5. Data Mapping
   - fields
   - source records
   - canonical model
   - system of record
   - legal basis
   - purpose
   - retention
   - classification
   - export eligibility

6. Payment Control
   - payment files
   - recipient registry
   - decision-payment matching
   - invoice-payment matching
   - reconciliation
   - risk rules

7. UBM Readiness
   - request intake
   - export proposal
   - legal/DPO review
   - maker-checker approval
   - receipt handling
   - notification handling
   - evidence chain

8. Go-live
   - DPIA
   - PUB/DPA
   - security review
   - RLS tests
   - SSO test
   - backup test
   - restore test
   - accessibility review
   - exit export test
   - UBM mock request
   - UBM mock export
   - payment reconciliation test
   - final approval
